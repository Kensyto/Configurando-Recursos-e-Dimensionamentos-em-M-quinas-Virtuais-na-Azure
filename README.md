# Resumo do Lab: Configurando Recursos e Dimensionamentos em Máquinas Virtuais na Azure

Este repositório contém um resumo das lições aprendidas durante o desenvolvimento do lab na DIO sobre configuração e dimensionamento de Máquinas Virtuais (VMs) na Microsoft Azure.

## 🚀 O que aprendi

Durante este lab, explorei os principais conceitos e práticas para implementar e gerenciar máquinas virtuais na nuvem de forma eficiente e escalável. Abaixo estão os tópicos principais:

### 1. Criação e Configuração de VMs
- **Seleção de Imagem:** Escolha do sistema operacional (Windows ou Linux) e versões específicas.
- **Tipos de Instância:** Entendimento das diferentes séries de VMs (A, D, E, F, etc.) otimizadas para diversos cenários como computação geral, memória ou GPU.

### 2. Dimensionamento (Scalability)
- **Escalabilidade Vertical:** Alterar o tamanho da VM (CPU/RAM) para lidar com mudanças na carga de trabalho.
- **Escalabilidade Horizontal (Virtual Machine Scale Sets):** Adicionar ou remover instâncias automaticamente com base na demanda, garantindo alta disponibilidade e otimização de custos.

### 3. Disponibilidade e Resiliência
- **Conjuntos de Disponibilidade (Availability Sets):** Proteção contra falhas de hardware e atualizações de infraestrutura.
- **Zonas de Disponibilidade (Availability Zones):** Isolamento de falhas em nível de datacenter dentro de uma região.

### 4. Redes e Segurança
- **Virtual Networks (VNet):** Configuração de sub-redes e isolamento de rede.
- **Grupos de Segurança de Rede (NSG):** Definição de regras de entrada e saída para controlar o tráfego de rede (Firewall).
- **IP Público vs. Privado:** Gerenciamento de acessibilidade externa.

### 5. Armazenamento (Storage)
- **Discos Gerenciados:** Diferença entre Standard HDD, Standard SSD, Premium SSD e Ultra Disk.
- **Persistência de Dados:** Configuração de discos de dados adicionais e backup.

## 💡 Conclusão
O lab proporcionou uma visão prática de como a Azure facilita o gerenciamento de infraestrutura, permitindo que empresas foquem no desenvolvimento de aplicações enquanto a plataforma cuida da escalabilidade e disponibilidade dos recursos.
